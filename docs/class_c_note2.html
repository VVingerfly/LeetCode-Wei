Detailed Description

如果要删除的是一个指向派生类对象的基类指针，则需要虚析构函数，确保派生类的析构函数被调用。

Note
当我们delete动态分配的对象的指针时将执行析构函数。如果该指针指向继承体系中的某个类型， 则有可能出现指针的静态类型与被删除对象的动态类型不符的情况。例如，如果我们delete一个Base*类型的指针， 则该指针有可能实际指向了一个Derived类型的对象。如果这样的话，编译器就必须清楚它要执行的是Derived的析构函数， 和其他函数一样，我们通过在基类中将析构函数定义为虚函数以确保执行正确的析构函数版本：
class Base
{
public:
// 如果我们删除的是一个指向派生类对象的基类指针，则需要虚析构函数
virtual ~Base() = default; //动态绑定析构函数
};
调用的顺序：
  • 构造函数：先调用父类的，再调用子类的构造函数
  • 析构函数：先调用子类的，再调用父类的析构函数
See also
C++ Primer Chapter 15.7.1 虚析构函数
